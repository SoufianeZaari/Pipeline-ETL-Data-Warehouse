\section*{Resume executif}
\addcontentsline{toc}{section}{Resume executif}

Ce rapport documente la conception et l'implementation d'un systeme decisionnel complet pour
\textbf{Mexora Analytics}, une marketplace e-commerce marocaine basee a Tanger. L'objectif est
de transformer des donnees transactionnelles brutes en un entrepot de donnees analytique
PostgreSQL permettant de repondre en temps reel a des questions metier complexes.

\begin{infobox}[title=Perimetre du projet]
\begin{itemize}[leftmargin=*]
  \item \textbf{Source :} 4 fichiers bruts intentionnellement defectueux (50\,000 commandes, 1\,000 clients, 300 produits, referentiel geographique)
  \item \textbf{ETL :} Pipeline Python complet (extraction, 14+ regles de transformation, chargement PostgreSQL)
  \item \textbf{DWH :} Schema en etoile PostgreSQL : 1 table de faits, 5 dimensions, 3 vues materialisees
  \item \textbf{Dashboard :} 5 questions analytiques visualisees via Metabase
\end{itemize}
\end{infobox}

\subsection{Schema en etoile -- FAIT\_VENTES}

\subsubsection{Granularite justifiee}

\begin{infobox}[title=Granularite choisie]
Une ligne de \texttt{FAIT\_VENTES} represente \textbf{une commande individuelle} : un produit
precis, achete par un client precis, livre dans une ville precise, a une date precise. Cette
granularite permet d'agreger le CA sur toutes les dimensions (produit, client, region, periode)
et de calculer le taux de retour par produit.
\end{infobox}

\subsection{Slowly Changing Dimensions (SCD)}

\subsubsection{Cas 1 --- DIM\_PRODUIT : changement de categorie (SCD Type 2)}

Un produit peut migrer de \textit{Telephones} vers \textit{Smartphones} suite a une refonte du
catalogue. Le projet choisit \textbf{SCD Type 2} : les colonnes
\texttt{date\_debut}, \texttt{date\_fin}, \texttt{est\_actif} permettent de conserver l'historique
analytique. Les ventes anterieures restent associees a l'ancienne version, assurant la coherence.

Les produits avec \texttt{"actif": false} dans la source JSON sont conserves dans
\texttt{dim\_produit} avec \texttt{est\_actif = FALSE} pour ne pas casser les jointures historiques.

\subsubsection{Cas 2 --- DIM\_CLIENT : changement de segment (SCD Type 2)}

Un client peut passer de \textit{Bronze} a \textit{Silver} quand son CA cumulé des 12 derniers mois
franchit 5\,000 MAD. Le projet applique egalement \textbf{SCD Type 2} sur \texttt{dim\_client},
permettant de savoir a quel segment un client appartenait lors de chaque achat passe.

\begin{alertbox}[title=Choix SCD Type 1 vs Type 2]
SCD Type 1 est utilise pour les \textbf{corrections de qualite} (villes mal orthographiees, emails
invalides) car ce sont des erreurs sans valeur historique. SCD Type 2 est reserve aux
\textbf{changements metier reels} (categorie produit, segment client) ou la tracabilite est critique.
\end{alertbox}

def transform_commandes(df: pd.DataFrame, regions: pd.DataFrame) -> pd.DataFrame:
    initial = len(df)
    # R1 - Doublons
    before = len(df)
    df = df.drop_duplicates(subset=['id_commande'], keep='last')
    logging.info("[TRANSFORM] R1 doublons: %s lignes supprimees", before - len(df))
    # R2 - Standardisation dates
    df['date_commande'] = pd.to_datetime(
        df['date_commande'], format='mixed', dayfirst=True, errors='coerce')
    invalid = int(df['date_commande'].isna().sum())
    df = df.dropna(subset=['date_commande'])
    logging.info("[TRANSFORM] R2 dates: %s invalides supprimes", invalid)
    # R3 - Harmonisation villes
    mapping_villes = charger_referentiel_villes(regions)
    df['ville_livraison'] = (df['ville_livraison'].str.strip().str.lower()
                             .map(mapping_villes).fillna('Non renseignee'))
    return df
\end{lstlisting}

\begin{lstlisting}[style=python, caption=build\_dim\_temps -- feries marocains et Ramadan]
def build_dim_temps(date_debut: str, date_fin: str) -> pd.DataFrame:
    dates = pd.date_range(start=date_debut, end=date_fin, freq='D')
    feries_maroc = {
        '2024-01-01',  # Nouvel An
        '2024-01-11',  # Manifeste de l'Independance
        '2024-05-01',  # Fete du Travail
        '2024-07-30',  # Fete du Trone
        '2024-11-18',  # Fete de l'Independance
        # ... + annees 2025-2026
    }
    ramadan_periods = [
        ('2024-03-10', '2024-04-09'),
        ('2025-03-01', '2025-03-30'),
        ('2026-02-18', '2026-03-19'),
    ]
    df['est_ferie_maroc'] = dates.strftime('%Y-%m-%d').isin(feries_maroc)
    df['periode_ramadan'] = False
    for debut, fin in ramadan_periods:
        masque = (df['date_complete'] >= debut) & (df['date_complete'] <= fin)
        df.loc[masque, 'periode_ramadan'] = True
    return df  # 2 557 lignes (2020-01-01 -> 2026-12-31)
\end{lstlisting}

\begin{lstlisting}[style=python, caption=calculer\_segments\_clients -- Gold/Silver/Bronze]
def calculer_segments_clients(commandes: pd.DataFrame) -> pd.DataFrame:
    """CA 12 mois: Gold >= 15 000 MAD, Silver >= 5 000 MAD, Bronze < 5 000 MAD"""
    commandes['montant_ttc'] = (commandes['quantite'].astype(float)
                                * commandes['prix_unitaire'].astype(float))
    ca = commandes.groupby('id_client')['montant_ttc'].sum().reset_index()
    ca['segment_client'] = pd.cut(ca['montant_ttc'],
        bins=[-1, 4999.99, 14999.99, float('inf')],
        labels=['Bronze', 'Silver', 'Gold']
    ).astype(str)
    return ca
\end{lstlisting}

\subsection{Phase Load --- PostgreSQL}

\begin{lstlisting}[style=python, caption=charger\_dimension et charger\_faits (loader.py)]
def charger_dimension(df, table_name, engine, if_exists='replace'):
    """TRUNCATE CASCADE + INSERT -- preserve les FK PostgreSQL."""
    with engine.connect() as conn:
        conn.execute(text(
            f"TRUNCATE TABLE dwh_mexora.{table_name} RESTART IDENTITY CASCADE"
        ))
        conn.commit()
    df.to_sql(table_name, engine, schema='dwh_mexora',
              if_exists='append', index=False, method='multi', chunksize=1000)
    logging.info("[LOAD] %s: %s lignes chargees", table_name, len(df))

def charger_faits(df, engine):
    """Truncate + bulk INSERT pour FAIT_VENTES."""
    with engine.connect() as conn:
        conn.execute(text(
            "TRUNCATE TABLE dwh_mexora.fait_ventes RESTART IDENTITY CASCADE"
        ))
        conn.commit()
    df.to_sql('fait_ventes', engine, schema='dwh_mexora',
              if_exists='append', index=False, method='multi', chunksize=5000)
    logging.info("[LOAD] fait_ventes: %s lignes chargees", len(df))
\end{lstlisting}

def run_pipeline() -> dict:
    # ---- PHASE EXTRACT ----
    df_commandes_raw = extract_commandes(ACADEMIC_RAW_DIR / 'commandes_mexora.csv')
    df_produits_raw  = extract_produits(ACADEMIC_RAW_DIR / 'produits_mexora.json')
    df_clients_raw   = extract_clients(ACADEMIC_RAW_DIR / 'clients_mexora.csv')
    df_regions       = extract_regions(ACADEMIC_RAW_DIR / 'regions_maroc.csv')
    # ---- PHASE TRANSFORM ----
    df_commandes = transform_commandes(df_commandes_raw, df_regions)
    df_clients   = transform_clients(df_clients_raw)
    df_produits  = transform_produits(df_produits_raw)
    dim_temps   = build_dim_temps('2020-01-01', '2026-12-31')   # 2 557 lignes
    dim_client  = build_dim_client(df_clients, df_commandes, df_regions)  # 898
    dim_produit = build_dim_produit(df_produits)                # 300
    dim_region  = build_dim_region(df_regions)                  # 12
    dim_livreur = build_dim_livreur(df_commandes)               # 6
    fait_ventes = build_fait_ventes(df_commandes, dim_temps,
                   dim_client, dim_produit, dim_region, dim_livreur)  # 36 203
    # ---- PHASE LOAD (PostgreSQL) ----
    engine = get_pg_engine()
    for name, df in [('dim_temps',dim_temps),('dim_client',dim_client),
                     ('dim_produit',dim_produit),('dim_region',dim_region),
                     ('dim_livreur',dim_livreur)]:
        charger_dimension(df, name, engine)
    charger_faits(fait_ventes, engine)
    return {'fait_ventes': len(fait_ventes), 'duree_secondes': 12}
\end{lstlisting}

\subsubsection{Vue 1 --- mv\_ca\_mensuel (914 lignes)}

\begin{lstlisting}[style=sql, caption=Vue materialisee -- CA mensuel par region et categorie]
CREATE MATERIALIZED VIEW reporting_mexora.mv_ca_mensuel AS
SELECT t.annee, t.mois, t.libelle_mois, t.periode_ramadan,
       r.region_admin, r.zone_geo, p.categorie,
       SUM(f.montant_ttc) AS ca_ttc,
       COUNT(DISTINCT f.id_commande) AS nb_commandes,
       SUM(f.quantite_vendue)        AS volume_vendu,
       ROUND(SUM(f.montant_ttc)
         / NULLIF(COUNT(DISTINCT f.id_commande), 0), 2) AS panier_moyen,
       COUNT(DISTINCT f.id_client)   AS nb_clients_actifs
FROM dwh_mexora.fait_ventes f
JOIN dwh_mexora.dim_temps   t ON f.id_date    = t.id_date
JOIN dwh_mexora.dim_region  r ON f.id_region  = r.id_region
JOIN dwh_mexora.dim_produit p ON f.id_produit = p.id_produit_sk
WHERE f.statut_commande = 'livre'
GROUP BY t.annee, t.mois, t.libelle_mois, t.periode_ramadan,
         r.region_admin, r.zone_geo, p.categorie
WITH DATA;
CREATE INDEX ON reporting_mexora.mv_ca_mensuel(annee, mois);
CREATE INDEX ON reporting_mexora.mv_ca_mensuel(region_admin);
\end{lstlisting}

\subsubsection{Q1 --- Evolution du CA par region}

\begin{lstlisting}[style=sql, caption=CA mensuel avec comparaison N-1]
WITH ca_mensuel AS (
    SELECT annee, mois, region_admin, SUM(ca_ttc) AS ca_total
    FROM reporting_mexora.mv_ca_mensuel
    GROUP BY annee, mois, region_admin
)
SELECT ca_total AS ca_mois_actuel,
       LAG(ca_total) OVER (PARTITION BY region_admin
                           ORDER BY annee, mois) AS ca_precedent,

             / NULLIF(LAG(ca_total) OVER (...), 0) * 100, 2) AS evolution_pct
FROM ca_mensuel ORDER BY annee DESC, mois DESC;

\subsubsection{Q3 --- Panier moyen par segment client}

\begin{lstlisting}[style=sql, caption=Panier moyen et part de CA par segment]
SELECT c.segment_client,
       COUNT(DISTINCT f.id_vente) AS nb_commandes,
       ROUND(SUM(f.montant_ttc)
             / COUNT(DISTINCT f.id_vente)::decimal, 2)  AS panier_moyen,
       ROUND(SUM(f.montant_ttc) * 100.0
             / SUM(SUM(f.montant_ttc)) OVER (), 2)       AS pct_ca
FROM dwh_mexora.fait_ventes f
JOIN dwh_mexora.dim_client c ON f.id_client = c.id_client_sk
WHERE f.statut_commande = 'livre' AND c.est_actif = TRUE
GROUP BY c.segment_client ORDER BY panier_moyen DESC;
\end{lstlisting}

\subsubsection{Q4 --- Taux de retour par categorie}

\section{Conclusion}

Ce projet demontre la mise en oeuvre complete d'un systeme decisionnel d'entreprise, depuis
l'analyse des besoins jusqu'au dashboard analytique, en respectant les 4 etapes du cahier de
charge.

\begin{infobox}[title=Bilan technique]
\begin{itemize}[leftmargin=*]
  \item \textbf{Etape 1 :} Modelisation en etoile rigoureuse, 5 dimensions, granularite justifiee,
    SCD Type 2 operationnel sur DIM\_PRODUIT et DIM\_CLIENT.
  \item \textbf{Etape 2 :} Pipeline Python structure \texttt{mexora\_etl/}, 14 regles documentees,
    logging complet, segmentation client automatisee, duree 12 s pour 50\,000 lignes source.
  \item \textbf{Etape 3 :} PostgreSQL 16 avec 3 schemas, 36\,203 faits charges, 3 vues
    materialisees fonctionnelles, 0 anomalie d'integrite sur 6 controles.
  \item \textbf{Etape 4 :} Dashboard Metabase, 5 questions analytiques avec KPIs et alertes
    visuelles (taux de retour, evolution CA, effet Ramadan).
\end{itemize}
\end{infobox}

\subsection*{Hypotheses et choix assumes}

\begin{enumerate}[leftmargin=*]
  \item \textbf{TVA 20\%} appliquee uniformement (\texttt{montant\_ttc = montant\_ht $\times$ 1.20})
    en l'absence de taux par categorie dans les donnees source.
  \item \textbf{Livreurs fictifs} (L001--L005 + inconnu $-1$) enrichis avec metadonnees statiques,
    faute d'un referentiel transporteur dans les donnees.
  \item \textbf{Statuts inconnus} exclus de la table de faits (contrainte CHECK PostgreSQL) et
    documentes dans les logs.
  \item \textbf{Instance PostgreSQL locale} sur port 5433 pour contournement des droits
    administrateur sur la machine academique.
\end{enumerate}
